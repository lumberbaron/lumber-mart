# references/

Spec templates for consistent documentation.

## Files

| File                        | What                              | When to read                                    |
| --------------------------- | --------------------------------- | ----------------------------------------------- |
| `user_story_template.md`    | Lightweight index entry format | Adding entries to user-stories.md index     |
| `story_template.md`         | Lightweight feature story (~100 lines) | Initial proposals, small features (1-2 components) |
| `architecture_template.md`  | System design template (~250 lines) | Multi-component features, new services, architectural changes |
| `component_template.md`     | Component spec structure (~180 lines) | Individual agents, services, APIs, UIs       |

## Template Selection

Use the decision tree in `../SKILL.md` to choose the right template:

- **story_template.md**: Small features, initial proposals
- **architecture_template.md**: Multi-component features, system design
- **component_template.md**: Single component deep-dive
